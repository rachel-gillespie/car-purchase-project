Secondly, after creating the 2 classes I began writing the methods in the CarPurchaseHelper.java file, below I detail what the methods do:

printWelcomeMessage (line 5 - 7)
- When called the method returns the message "Welcome to the Car Purchase Helper Program! Let's find the perfect car for you!". 

calculateHP (line 10 - 31)
A finance plan where you pay a deposit and then equal monthly payments until the full amount is paid. You own the car once the final payment is made.
- This method is used to calculate the monthly Hire Purchase payments. 
- It takes in carPrice, deposit, annualInterest and termYears from the user to calculate HP.
- amountToFinance is calculated by subtracting deposit from carPrice.
- monthlyInterestRate is calculated by dividing annualInterest by 100 and the answer by 12.
- totalMonths is calculated by multiplying termYears by 12.
- It then uses the formula (1 + r)^n.
- r is the monthlyInterestRate and n is the number of totalMonths.
- A for loop is used to calculate (1 + r)^n, it is then stored as a variable called factor.
- Factor, amountToFinance and monthlyInterestRate are then used to calculate the monthly payment using the formula, monthlyPayment = L * r * factor / (factor - 1).
- L is the amountToFinance and r is the monthlyInterestRate.

pcpResultMonth (line 34 - 56)
A type of car finance where you pay a deposit, followed by monthly payments over a fixed term. At the end, you can either return the car, pay a final balloon payment to own it, or trade it in.
- This method is used to calculate the monthly Personal Contract Plan payments.
- It takes the carPrice, deposit, annualInterest, termYears and gmfvPercent from the user to calculate PCP.
- gmfv is calculated by dividing gmfvPercent by 100 and then mulitplying carPrice by that answer.
- amountToFinance is calculated by subtracting deposit from carPrice and then subtracting gmfv.
- monthlyInterestRate is calculated by dividing annualInterest by 100 and the answer by 12.
- totalMonths is calculated by multiplying termYears by 12.
- It then uses the formula (1 + r)^n.
- r is the monthlyInterestRate and n is the number of totalMonths.
- A for loop is used to calculate (1 + r)^n, it is then stored as a variable called factor.
- Factor, amountToFinance and monthlyInterestRate are then used to calculate the monthly payment using the formula, monthlyPayment = L * r * factor / (factor - 1).
- L is the amountToFinance and r is the monthlyInterestRate.

fuelCostEstimator (line 58 - 65)
- This method is used to calculate weekly Fuel Cost.
- It takes the weeklyKilometers and fuelCost per litre from the user input to calculate weeklyFuelCost.
- weeklyFuelCost is calculated by dividing weekly kilometers by 100 multiplying the answer by 6 and then by fuelCost.
- Assuming the car uses 6L of fuel per 100km.

calculateMonthlyCost (line 67 - 74)
- This method is used to calculate monthly cost.
- It takes the carPrice and termMonths from the user input to calculate monthlyPayment.
- monthlyPayment is calculated by dividing carPrice by termMonths.

recommendCarType (line 76 - 87)
- This method is used to recommend a car type to the user.
- It takes the carRequiremnt from the user input.
- Based on the requirement entered it returns a recommended car.
- If the user doesn't enter a recognised requirement this message is returned "You did not input a valid requirement!".

returnToMenu (line 89 - 91) ***
- This method is used to bring the user back to the main menu.
- When called it checks to see if the user input is equal to yes while ignoring letter case and returns a true value.

printGoodbyeMessage (line 93 - 95)
- When called the method returns the message "We hope you had a successful experience! Goodbye!"

These methods can be called whenever needed in the Driver class, reducing repetition and making the code easier to read and maintain.
